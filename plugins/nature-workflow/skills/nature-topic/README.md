# `nature-topic` 技能

`nature-topic` 用于在检索导出之后、正式筛选与写作之前，对导出的 RIS/参考文献记录做专业整合、聚类与分析，
并从真实文献模式中提炼 3 个可写的综述候选选题，比较研究空白。它是 `nature-orchestrator` 编排链中被委托的
一个原子步骤：orchestrator `start` 本步，拿到产物后用 `complete <task_id> --evidence <产物路径>` 收尾。

本技能支持中文输入。用户可以直接提出“选题”“研究空白”“综述选题”“确定选题”“RIS 年份标题摘要分析”等需求；
分析说明默认按用户语言返回，选题标题保持中英双语。

## 功能

- 基于导出记录的年份、标题、摘要做整合、分类与分析，生成年份-标题-摘要分析表。
- 将记录聚类，识别拥挤热点、新兴机制/技术、争议点与未被充分综述的研究空白。
- 沿 稳妥 / 平衡 / 高冲击 给出 3 个候选选题，并标注 1 个推荐项。
- 每个候选强制定义：研究对象、核心机制/科学问题、综述切入角度、具体研究空白、目标分区适配。
- 作为决策步：只产出多候选，交由 orchestrator 按 decision 协议向用户呈现，不替用户拍板。
- 产物为分析表 + 候选选题的单个 `.md` 文件（无需 docx），路径可直接作为 evidence。

## 来源基础

融合自 SCI 工作流步骤 `04-SCI选题与研究空白识别`。保留其硬规则：候选选题必须来自真实导出文献模式而非关键词
想象；固定选题模式为 `研究对象 + 核心机制/关键变量 + 临床或科学问题 + 综述视角`；禁止宽泛标题（反例
`肠道菌群与肿瘤免疫治疗研究进展`）；本步是选题分析而非正式筛选，不得声称最终纳入篇数；不得编造具体论文、
作者、DOI、PMID、期刊指标或引用数；元数据缺失标 `需要人工核查`。伦理红线沿用 `_shared/core/ethics.md`。

## 文件结构

该技能采用 router/static-dynamic 结构：`SKILL.md` 是短路由，`manifest.yaml` 决定常驻 core 与按需 references。
领域逻辑全部在 `static/core`，没有内容轴。

```text
nature-topic/
├── SKILL.md                     # 短路由（原子步骤 + 决策步说明）
├── manifest.yaml                # always_load core + _shared/ethics + 按需 references
├── README.md
└── static/
    └── core/                    # 始终加载
        ├── stance.md            # 立场、固定选题模式、Prohibited/red-lines
        ├── workflow.md          # 编号执行步骤 + 每步产物
        └── output-contract.md   # 产物清单 + 交付格式（.md 决策表）
```

## 核心规则表

| 规则 | 要求 |
|---|---|
| 证据来源 | 候选选题只来自导出记录的年份-标题-摘要模式，不靠关键词想象或记忆 |
| 选题模式 | 固定为 研究对象 + 核心机制/关键变量 + 临床或科学问题 + 综述视角 |
| 禁止宽泛标题 | 反例 `肠道菌群与肿瘤免疫治疗研究进展`；无机制/无具体问题/无视角一律不合格 |
| 每候选必含 | 研究对象 / 核心机制或科学问题 / 综述切入角度（7 类之一）/ 具体研究空白 / 目标分区适配 |
| 切入角度 | 机制型 / 临床转化型 / 诊疗策略型 / 生物标志物型 / 技术方法型 / 争议整合型 / 未来方向型 |
| 三候选 | A 稳妥型 / B 平衡型 / C 高冲击型，标 1 个推荐项 |
| 决策步 | 只产出多候选，交 orchestrator 按 decision 协议呈现，不替用户选 |
| 不编造 | 具体论文/作者/DOI/PMID/期刊指标/引用数一律不得虚构 |
| 阶段边界 | 是选题分析，非正式筛选；不得声称最终纳入篇数 |
| 缺失标记 | 元数据不全标 `需要人工核查`；动态期刊信息（分区/IF/范围/APC/投稿规则）须联网核验 |
| 产物 | 单个 `.md`（分析表 + 候选选题 + 决策块），返回绝对路径作 evidence |
