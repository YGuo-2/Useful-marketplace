# `nature-screening` 技能

`nature-screening` 用于把一批原始文献导出记录，治理成可导入文献管理器的“综述文献库”：去重、按题名/摘要与主题的匹配程度做相关性排序、保留前 N（默认 500）篇强相关文献、按对标综述写作框架建立二级/三级分类，并生成 Zotero 可导入的结构与映射表。

该技能是 `nature-orchestrator` 编排流程中的一个原子步骤：orchestrator `start` 该步、委托本技能执行、拿到产物后用 `complete --evidence <产物路径>` 收口。它也可以单独运行，用于纯粹的文献筛选与分类。

本技能显式做到**跨学科**：医学、生命科学、工程、材料、化学、物理、计算机、环境、农业、管理、社科及交叉学科都在覆盖范围内，不把生物医学词汇硬套到非生医主题上。

## 功能

- 把 `.ris`/`.nbib`/`.bib`/`.csv`/`.txt` 等原始导出（PubMed / Web of Science / Scopus 或其他学科数据库）解析成统一筛选总表。
- 保守去重：先精确 DOI/PMID/PMCID/WoS 号，再有把握时做模糊匹配；不确定不合并、标“待人工核查”；不删除原始重复记录。
- 按题名/摘要与主题的匹配程度做相关性排序，给出四级相关性：`强相关`/`中等相关`/`弱相关/排除`/`待人工核查`。
- 保留前 N（默认 500）篇强相关文献；不足 N 时保留全部强相关并记录实际数量，绝不用弱相关凑数。
- 按对标综述写作框架建立二级/三级分类，每篇必须落到一个三级类或 `99-待人工核查`。
- 生成 Zotero 可导入结构：分类后 RIS/参考文献文件、导入映射表、集合结构清单。
- 输出筛选日志、去重报告、分类日志和质量核查清单。

## 来源基础

- 融合自 SCI从0-1workflow 的 `06-SCI文献筛选分类器`（阶段 3，综述专属步骤）。核心是“保留强相关 + 按对标框架分类”，而不是下载 PDF。
- 上游依据：已确认的综述主题与研究空白、最终检索式、对标综述写作框架与分类指导。
- 原始导出文件视为只读；所有分类后文件都是派生产物，原始文件保持不变。

## 文件结构

该技能采用 router/static 结构：`SKILL.md` 负责短路由，`manifest.yaml` 加载常驻 core 与共享伦理红线。`nature-screening` 是线性工作流，没有内容轴。

```text
nature-screening/
├── SKILL.md                     # 短路由：说明本技能是 nature-orchestrator 委托的原子步骤
├── manifest.yaml                # always_load core + ../_shared/core/ethics.md
├── README.md
└── static/
    └── core/                    # 始终加载
        ├── stance.md            # 跨学科立场、来源层级、禁止事项/红线
        ├── workflow.md          # 编号执行步骤 + 每步产物
        └── output-contract.md   # 产物文件清单、文件夹树、用户报告格式
```

## 适用场景

- 拿到一批检索导出后、进入全文精读之前，先做去重、排序、筛选和分类。
- 为综述建立强相关文献库，并规划 Zotero 二级/三级集合结构。
- 把跨学科的原始记录治理成可交接的分类映射表与文件夹树。
- 判断某篇文献属于强相关、中等相关、弱相关-排除，还是待人工核查。

## 核心规则

| 规则 | 说明 |
|---|---|
| 跨学科 | 覆盖医/生/工/材/化/物/CS/环境/农/管/社科/交叉；禁把生物医学词汇硬套到非生医主题 |
| 从泛到精 | 学科方向 → 关键词/检索式 → 确认主题/研究空白 → 对标框架，再判定每篇 |
| 保守去重 | 先精确 DOI/PMID/PMCID/WoS 号，再模糊；不确定标“待人工核查”不合并；不删原始重复记录 |
| 四级相关性 | `强相关`/`中等相关`/`弱相关/排除`/`待人工核查`，每篇附题名/摘要判定依据 |
| 前 N 保留 | 默认保留前 500 篇强相关；不足则保留全部并记录实际数；不得用弱相关凑数 |
| 全员分类 | 11 个默认二级类（01-背景与概念基础 … 10-综述与指南背景文献 / 99-待人工核查）；每篇落到一个三级类或 99 |
| 原始只读 | 原始导出文件保持不变；分类后 RIS 为派生文件 |
| 层级权威 | 交接以映射表 + 文件夹树为准，不承诺任何导入器都能自动复原集合层级 |
| 不编造 | 不编造 DOI/PMID/作者/期刊/元数据；缺失留空或标“缺失/需要人工核查” |
| 动态必核 | 期刊范围/IF/分区/APC/投稿规则/许可等动态信息必须联网核验，不得凭记忆 |

## 交付格式

产物均为 `.csv`/`.ris`/`.md` 表格与清单，不产出 `.docx`。可作为 orchestrator `--evidence` 的典型产物是 `输出/XXXXX综述文献库/二级三级分类表.csv` 或 `输出/纳入文献列表.csv`。
