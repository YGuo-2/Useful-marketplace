# `nature-benchmark` 技能

`nature-benchmark` 用于综述（review）写作：从上游文献库和已确认的综述主题中，筛选出与本综述最匹配、期刊影响力和文章质量最高的约 10 篇综述，建立“对标综述文献库”并导入 Zotero，深度学习它们的框架、写作逻辑、写作风格和图表排版，并把学习结果转成给下一步文献筛选/分类用的分类指导框架。

该技能是被 `nature-orchestrator` 委托的原子步骤（综述文体模板第 4 步 `benchmark`，阶段 3）。orchestrator 会 `start` 该步、把步骤简报交给本技能，拿到产物后用 `complete --evidence <产物路径>` 收口。因此本技能的产物会明确给出可作 evidence 的文件路径。

## 功能

- 从上游筛选/导出的文献库中识别综述文献（按文献类型、题录/摘要信号、数据库标签、MeSH、文献类型字段、期刊元数据判定）。
- 用透明打分表排序：先主题匹配，再期刊/文章质量。
- 选出约 10 篇对标综述；不足 10 篇时保留最大可得数并记录原因，不用弱匹配凑数。
- 生成对标综述排序表（CSV）与筛选理由（MD），导出“对标综述文献库 RIS”。
- 按优先级导入 Zotero：先用 Zotero 插件/连接器 → 失败先记录原因再用 Computer Use → 仍不行给方框手动提示。
- 等用户在 Zotero 中获取 PDF 并回复后，只对已获得的 PDF 做全文级深度阅读。
- 只学结构/叙事/章节节奏/证据组织/图表策略，禁止复制标题、小标题、段落、图表和观点。
- 产出深度学习报告、拟定标题与写作思路，以及给下一步分类的二级/三级分类指导框架。

## 来源基础

本技能融合自 SCI 从 0-1 工作流的 `05-SCI建立综述对标库并深度学习`（综述专属阶段），并按 nature-workflow 的 router / static-dynamic 规范重构为常驻 core + 共享伦理红线。

- 上游产物：筛选/导出的文献库，以及已确认的综述主题、选题理由和研究空白。
- 结构化题录元数据：Crossref、PubMed/NCBI E-utilities、DOI 元数据、数据库标签。
- 官方出版商/期刊页面：用于核对期刊身份与当前指标。
- 合法获得的对标综述 PDF：用于全文级结构学习。

动态信息（期刊范围、影响因子、分区、CiteScore、APC、投稿规则、许可）必须联网核验，不得凭记忆；无法核实的标 `需要人工核查`。

## 文件结构

该技能采用 router / static-dynamic 结构：`SKILL.md` 负责短路由，`manifest.yaml` 决定常驻内容。`nature-benchmark` 是线性工作流，没有内容轴，因此由 core 常驻内容加共享伦理红线组成。

```text
nature-benchmark/
├── SKILL.md                     # 短路由（英文），说明被 orchestrator 委托
├── manifest.yaml                # always_load core + 共享 ethics
├── README.md
└── static/
    └── core/                    # 始终加载
        ├── stance.md            # 立场 + 红线/禁止事项 + 来源层级
        ├── workflow.md          # 编号执行步骤 + 每步产物
        └── output-contract.md   # 产物清单 + 交付格式 + 质量门槛
```

（伦理红线来自 `../_shared/core/ethics.md`，由 `manifest.yaml` 的 `always_load` 常驻加载。）

## 核心规则

| 规则 | 说明 |
|---|---|
| 选 10 篇的排序 | 先主题匹配，再期刊/文章质量 |
| 不足 10 篇 | 保留最大可得数并记录原因，不用弱匹配凑数 |
| Zotero 导入优先级 | 插件/连接器 → 失败记原因后 Computer Use → 再不行给方框手动提示 |
| PDF 获取 | 等用户回复；只对已获得 PDF 做全文级分析 |
| 学习边界 | 只学结构/叙事/风格/排版策略，禁止复制标题/小标题/段落/图表/观点 |
| 期刊指标 | 联网核验，不可核实的标 `需要人工核查` |
| 不编造 | 文献/DOI/PMID/作者/期刊/年份/摘要/指标/授权状态一律不编造，缺失留空或标 `缺失/需要人工核查` |
| 禁止事项 | 禁 Sci-Hub/盗版/绕过付费墙/批量非法下载；禁索取 Zotero 或机构凭据；禁谎称已读取 PDF |
| 不越界 | 不做大规模原文筛选、不做正式全库二/三级分类、不做全文证据提取、不写正文 |

## 交付物

产物为对标库 RIS + 分析/学习报告（`.md`/`.csv`），无需 docx：

```text
输出/
├── 对标综述文献库RIS/对标综述文献库.ris   # 对标综述库
├── 对标综述排序表.csv
├── 对标综述筛选理由.md
├── Zotero导入记录.md
├── 对标综述PDF状态表.csv
├── 对标综述整理素材库.md                 # 有全文阅读时才生成
├── 对标综述深度学习报告.md               # evidence：benchmark-corpus report
├── 拟定标题与写作思路.md
└── 下一步文献分类指导框架.md             # evidence：classification guide
```

交给 orchestrator 的 evidence：`对标综述深度学习报告.md`（对标语料报告）与 `下一步文献分类指导框架.md`（分类指导）。
